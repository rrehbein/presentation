O - Open / Closed

Objects should be open to extension, but closed to modification

O - Open / Closed

Meyer's Open / Closed Principle

  • Object Oriented Software Construction - 1988
    • Implementation can be reused through inheritance
    • Interface specifications need not be

O - Open / Closed

Polymorphic Open / Closed Principle

  • "abstracted interfaces" - During 1990s
  • Same interface
  • Different implementation

L - Liskov's Substitution

Objects of the same type should be interchangeable

L - Liskov's Substitution

Type

  • Class
  • Parent class
  • Interface

L - Liskov's Substitution

Interchangeable

  • An object's methods can be called in the same way as another of its type
    • Accepts the same types of parameters
    • Returns the same types of values

L - Liskov's Substitution

Objects of the same type should be interchangeable
$service->setCache($memCache);
$service->setCache($fileCache);

L - Liskov's Substitution

Objects of the same type should be interchangeable

D - Dependency Inversion

Applications should depend upon abstractions, not concrete instances of an object

D - Dependency Inversion

Abstractions

  • An abstract class
  • An interface
  • A 'type' of object

D - Dependency Inversion

Inversion

  • A reversal of the normal order
  • In the context of SOLID, objects should depend on abstractions

Injection

  • The act of injecting or throwing in forcibly
  • Not actually part of SOLID

D - Dependency Inversion

class MyLoggingCache implements PSR\Cache {
    public function __construct(
        PSR\Cache $realCache
    ) {
        $this->cache = $realCache;
        // UGLY: Depends on a concrete logger
        $this->logger = new MyFileLogger();
    }

D - Dependency Inversion

class MyLoggingCache implements PSR\Cache {
    public function __construct(
        PSR\Cache $realCache, PSR\Logger $logger
    ) {
        $this->cache = $realCache;
        // The logger is injected in
        $this->logger = $logger;
    }

D - Dependency Inversion

Service Locators vs DIC

  • Good things about SL over DIC
    • Lazy creation
  • Bad things about SL over DIC
    • Unit Testing needs one more mock
    • Multiple instances of a class means multiple SLs

D - Dependency Inversion

SL vs Dependency Injection Containers

  • Good things about DIC over SL
    • Simpler unit tests
    • Allows alternate instances of a class
  • Bad things about DIC over SL
    • Object creation
